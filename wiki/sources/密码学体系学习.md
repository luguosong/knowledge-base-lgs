## 核心要点

### 基础篇：密码学原语（Primitives）

1. **Java 密码学架构（JCA/JCE）**：Provider 机制实现「一套接口，多种实现」，Bouncy Castle 作为主要第三方 Provider；SecureRandom 质量直接决定密钥安全性
2. **对称加密**：AES 是性能基石，ECB 模式因模式泄露已被废弃；GCM 是当前推荐的 AEAD 模式（CTR 加密 + GHASH 认证）；ChaCha20-Poly1305 在无 AES-NI 的移动端表现更优；nonce 重用是流密码的致命错误
3. **哈希与完整性**：SHA-2（Merkle-Damgård 构造）和 SHA-3（Sponge 构造）；HMAC 提供带密钥的消息认证；KDF 从弱密码派生强密钥；Merkle 树高效验证海量数据
4. **随机数与密钥派生**：PRNG 的熵源质量决定系统安全上限；HKDF 是标准密钥派生函数；阈值密码学（Shamir 秘密分享）用于密钥分片
5. **基于密码的密钥生成**：PBKDF2、scrypt、Argon2 依次增强了抗 GPU/ASIC 暴力破解能力

### 协议篇：非对称与公钥基础设施

6. **非对称加密与混合加密**：RSA-OAEP 解决了 PKCS#1 v1.5 的 Bleichenbacher 攻击；KEM-DEM 范式（非对称传递密钥 + 对称加密数据）；ECIES 基于椭圆曲线的完整混合加密方案
7. **密钥交换**：Diffie-Hellman 及其椭圆曲线变体 ECDH（X25519/X448）；密钥确认防范中间人攻击；前向保密依赖临时密钥交换
8. **数字签名**：DSA/ECDSA 基于离散对数；EdDSA（Ed25519）天然确定性且抗侧信道；RSA-PSS 有可证明安全性；SM2 国密签名；签名延展性攻击（MtGox 事件）
9. **证书与 PKI**：X.509 证书链、CRL/OCSP 吊销检查、证书路径验证；CA 信任链失败案例
10. **密钥存储**：KeyStore 类型对比（JKS/PKCS12/BCFKS）；keytool 工具；硬件存储引子

### 应用篇：真实世界协议

11. **TLS**：TLS 1.2 到 1.3 的重大改进（2-RTT → 1-RTT、强制 AEAD、强制前向保密）；密码套件协商；JSSE 框架（SSLContext/TrustManager）；QUIC 内嵌 TLS 1.3
12. **端到端加密**：Signal 协议（X3DH + Double Ratchet）实现前向与后向安全；PGP 邮件加密为何失败；信任根问题
13. **用户认证**：OPAQUE（aPAKE）、TOTP、WebAuthn/FIDO2、CPace
14. **CMS 与 S/MIME**：加密消息语法、电子邮件加密标准、时间戳协议
15. **OpenPGP**：PGP 协议、密钥环管理、Web of Trust 的局限性
16. **加密货币与 BFT 共识**：Bitcoin 工作原理、UTXO 模型、PoW、DiemBFT
17. **硬件密码学**：HSM、TPM、Secure Element、TEE；白盒密码的局限性；侧信道与常时间编程

### 前沿篇

18. **后量子密码**：Shor 算法威胁非对称加密（RSA/ECC），Grover 算法只需翻倍对称密钥长度；NIST 标准 ML-KEM（FIPS 203）、ML-DSA（FIPS 204）、SLH-DSA（FIPS 205）；格密码（LWE/MLWE）原理；混合方案（ECDH + ML-KEM）是过渡期最佳实践
19. **下一代密码学**：MPC（多方安全计算/PSI）、FHE（全同态加密）、ZKP（zk-SNARKs/zk-STARKs）

### 附录

20. **ASN.1 基础**：密码学协议的数据编码基础（DER/BER 规则）
21. **密码学失败案例集**：83% 的漏洞源于 API 误用而非算法弱点；涵盖 Sony PS3、Debian OpenSSL、Padding Oracle、Heartbleed 等经典案例

## Java 实现聚焦

- **JCA/JCE**：`Cipher`、`Signature`、`MessageDigest`、`KeyGenerator`、`KeyPairGenerator` 等引擎类
- **Bouncy Castle**：JCA Provider 注册（`Security.addProvider(new BouncyCastleProvider())`）；轻量级 API 用于 PQC 算法（`MLKEMGenerator`/`MLKEMExtractor`）
- **JSSE**：`SSLContext` → `SSLSocket` 的 TLS 编程模型

## 安全实践启示

- 不要自制密码算法，不要自行实现标准算法——使用经过审计的库
- 90% 的密码学漏洞不在算法本身，而在协议组合与误用
- nonce/IV 管理是 AEAD 模式最关键的安全参数
- 后量子迁移紧迫性来自"先收集，后破解"攻击模式
