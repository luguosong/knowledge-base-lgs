系统学习 GoF 23 种经典设计模式（含简单工厂共 24 种），按创建型、结构型、行为型三大类组织。内容从面向对象四大特性和类间六种关系出发，以 SOLID 原则为理论基石，深入分析每种模式的意图、结构和适用场景。特别关注 JDK、Spring、MyBatis、Servlet/Tomcat 四大框架中的设计模式实际应用，以及 Java 8 函数式编程如何简化传统模式实现。

## 核心要点

- 系统学习了 GoF 23 种经典设计模式（+ 简单工厂共 24 种），按创建型 5 种、结构型 7 种、行为型 11 种三大类组织
- 设计模式建立在面向对象四大特性（抽象、封装、继承、多态）之上，理解 OOP 基因是掌握模式的前提
- SOLID 五大原则（SRP、OCP、LSP、ISP、DIP）是设计模式背后的"道理"，模式是"套路"
- 深入分析了 JDK、Spring、MyBatis、Servlet/Tomcat 四个主流框架中的设计模式实际应用
- Java 8 函数式接口将行为型模式"语言化"，lambda 简化了策略、命令等模式的仪式感

## 详细摘要

### OOP 基础

设计模式生长在面向对象范式之上。核心内容包括：

- **四大特性**：抽象（只建模当前场景所需部分）、封装（隐藏内部实现细节）、继承（子类复用父类代码）、多态（运行时根据实际类型调用正确方法）
- **类间六种关系**（由弱到强）：依赖、关联、聚合、组合、实现、继承
- **UML 类图**：读懂设计模式图示的通用语言
- **设计模式的 OOP 基因**：创建型依赖封装，结构型依赖组合关系，行为型依赖多态；23 种模式中只有模板方法以继承为核心

### 设计原则

设计模式是"套路"，设计原则是"道理"。核心原则包括：

- **封装变化**：几乎所有模式都是这一思想在不同场景下的具体实现
- **针对接口编程**：调用方依赖抽象，不直接依赖具体实现类
- **松耦合设计**：对象之间相互知道的信息越少越好
- **组合优于继承**：多维度变化时避免继承爆炸
- **SOLID 原则**：SRP（单一职责）、OCP（开闭）、LSP（里氏替换）、ISP（接口隔离）、DIP（依赖倒置）
- **其他原则**：DRY、KISS、YAGNI、迪米特法则、合成复用原则、好莱坞原则

### 创建型模式（5 + 1 种）

关注对象的创建过程，当直接 `new` 带来耦合或复杂度时提供灵活方案：

- 单例（Singleton）：私有构造函数 + 静态方法
- 简单工厂（Simple Factory）：静态工厂方法 + switch（非 GoF）
- 工厂方法（Factory Method）：继承 + 重写工厂方法
- 抽象工厂（Abstract Factory）：接口 + 多个工厂方法
- 建造者（Builder）：链式调用 + `build()`
- 原型（Prototype）：`clone()` 深拷贝

### 结构型模式（7 种）

关注类和对象的组合方式，保持结构的灵活性和高效性：

- 外观（Facade）：聚合子系统，暴露高层接口
- 适配器（Adapter）：包装不兼容类，实现目标接口
- 代理（Proxy）：实现相同接口 + 持有真实对象
- 装饰器（Decorator）：实现相同接口 + 持有被装饰对象
- 桥接（Bridge）：组合代替继承，两维度各自扩展
- 组合（Composite）：叶子和容器统一接口
- 享元（Flyweight）：分离内/外部状态，工厂缓存共享对象

### 行为型模式（11 种）

关注对象之间的职责分配与通信：

- 模板方法（Template Method）：继承 + 钩子方法
- 策略（Strategy）：接口抽象算法，运行时注入
- 状态（State）：将状态抽象为独立类
- 命令（Command）：命令接口 + execute()
- 责任链（Chain of Responsibility）：处理者持有 next 引用
- 观察者（Observer）：发布-订阅，维护监听者列表
- 中介者（Mediator）：组件只与中介者通信
- 迭代器（Iterator）：hasNext()/next() 接口
- 备忘录（Memento）：快照对象 + 历史栈
- 访问者（Visitor）：双分派，accept() + visit()
- 解释器（Interpreter）：语法规则转类，组合构成 AST

### 框架中的设计模式应用

**JDK**：策略模式（`Comparator`、函数式接口）、模板方法（`AbstractList` 骨架实现）、装饰器（`java.io` FilterInputStream 链 + `Collections.unmodifiableXxx`）、命令模式（`Runnable`/`FutureTask`）、组合模式（`java.awt.Component` 树）等。

**Spring**：工厂模式（`BeanFactory`）、单例（IoC 容器管理）、代理（Spring AOP）、策略（JDK/CGLIB 动态代理选择）、适配器（`HandlerAdapter` 统一 Controller 接口）、观察者（`ApplicationEvent`）、模板方法（`JdbcTemplate` 回调实现）、装饰器（`TransactionAwareCacheDecorator`）、组合（`CompositeCacheManager`）、职责链（`HandlerExecutionChain`）、解释器（SpEL）共 11 种模式。

**MyBatis**：从 SqlSession 初始化到 Cache 增强，识别 10 种设计模式。

**Servlet/Tomcat**：责任链（Filter 链）、模板方法（Servlet 生命周期）、外观模式等。
