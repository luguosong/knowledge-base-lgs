## 定义

设计模式（Design Patterns）是软件工程领域经过反复验证的通用解决方案——不是可直接运行的代码，而是解决特定类型问题的"套路模板"。由 GoF（Gang of Four：Erich Gamma、Richard Helm、Ralph Johnson、John Vlissides）在 1994 年的经典著作中系统整理，共 23 种经典模式，分为创建型、结构型、行为型三大类。

## 学习价值

- **提效**：遇到一类问题时快速识别"套路"，直接应用成熟方案
- **沟通**：用模式名称精确沟通，减少团队歧义
- **读懂框架**：Spring、JDK、MyBatis 等大量使用设计模式
- **平衡设计**：避免"过度设计"和"设计不足"两个极端

## 三大分类

### 创建型模式（5 种 + 简单工厂）

关注对象的创建过程，当直接 `new` 带来耦合或复杂度时提供灵活方案。

| 模式 | 一句话说明 | 核心手段 |
|------|-----------|---------|
| 单例（Singleton） | 全局只有一个实例 | 私有构造函数 + 静态方法 |
| 简单工厂（Simple Factory） | 工厂类集中管理创建逻辑 | 静态工厂方法 + switch |
| 工厂方法（Factory Method） | 子类决定创建哪个具体类 | 继承 + 重写工厂方法 |
| 抽象工厂（Abstract Factory） | 创建一族相关对象 | 接口 + 多个工厂方法 |
| 建造者（Builder） | 分步骤组装复杂对象 | 链式调用 + `build()` |
| 原型（Prototype） | 克隆现有对象 | `clone()` 深拷贝 |

### 结构型模式（7 种）

关注如何将类和对象组合成更大的结构，保持灵活性和高效性。

| 模式 | 一句话说明 | 核心手段 |
|------|-----------|---------|
| 外观（Facade） | 为复杂子系统提供简单接口 | 聚合子系统，暴露高层接口 |
| 适配器（Adapter） | 让不兼容的接口协同工作 | 包装不兼容类，实现目标接口 |
| 代理（Proxy） | 控制对目标对象的访问 | 实现相同接口 + 持有真实对象 |
| 装饰器（Decorator） | 运行时动态添加功能 | 实现相同接口 + 持有被装饰对象 |
| 桥接（Bridge） | 抽象与实现分离，独立扩展 | 组合代替继承，两维度各自扩展 |
| 组合（Composite） | 用树形结构表示整体-部分关系 | 叶子和容器统一接口 |
| 享元（Flyweight） | 共享细粒度对象减少内存占用 | 分离内/外部状态，工厂缓存共享对象 |

适配器、装饰器、代理三者都是"包装"对象，区别在意图：适配器改变接口以兼容，装饰器接口不变动态增强功能，代理接口不变控制访问。

### 行为型模式（11 种）

关注对象之间的职责分配与通信。

| 模式 | 一句话说明 | 核心手段 |
|------|-----------|---------|
| 模板方法（Template Method） | 父类定义骨架，子类填充细节 | 继承 + 钩子方法 |
| 策略（Strategy） | 定义一族算法，可互相替换 | 接口抽象算法，运行时注入 |
| 状态（State） | 对象行为随状态变化而改变 | 将状态抽象为独立类 |
| 命令（Command） | 把请求封装成对象 | 命令接口 + execute() |
| 责任链（Chain of Responsibility） | 请求沿处理者链传递 | 处理者持有 next 引用 |
| 观察者（Observer） | 一变多通知 | 发布-订阅，维护监听者列表 |
| 中介者（Mediator） | 集中管理对象间的通信 | 组件只与中介者通信 |
| 迭代器（Iterator） | 顺序访问集合，不暴露内部结构 | hasNext()/next() 接口 |
| 备忘录（Memento） | 保存并恢复对象状态 | 快照对象 + 历史栈 |
| 访问者（Visitor） | 在不修改类的前提下添加新操作 | 双分派，accept() + visit() |
| 解释器（Interpreter） | 定义语言的文法，并实现解释 | 语法规则转类，组合构成 AST |

## 模式背后的 OOP 基因

| 模式分类 | 核心 OOP 基石 | 典型体现 |
|---------|------------|---------|
| 创建型 | 封装 | 工厂方法把 `new` 封装到子类，调用方不知具体类型 |
| 结构型 | 组合关系 | 装饰器、代理、适配器通过持有目标对象引用包装行为 |
| 行为型 | 多态 | 策略、命令、状态等把行为抽象为接口，运行时多态分派 |

23 种模式中只有模板方法以继承为核心机制，其余几乎都通过接口/组合实现可扩展性，印证了"组合优于继承"。

## 与其他概念的关系

- [[SOLID原则]]：设计模式背后的理论基石，每个模式都可追溯到至少一条原则
- [[Spring Framework]]：设计模式在工业界最密集的应用案例之一，使用了 11 种模式
