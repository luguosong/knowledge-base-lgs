![OAuth 2.0 logo](https://oauth.net/images/oauth-2-sm.png) OAuth 2.0 是行业标准的授权协议。它专注于为客户端开发者提供简洁性，同时为 Web 应用、桌面应用、移动设备和客厅设备提供特定的授权流程 (Authorization Flow)。该规范及其扩展正在 [IETF OAuth Working Group](https://datatracker.ietf.org/wg/oauth/about/) 中开发。

[OAuth 2.1](https://oauth.net/2.1/) 是一项正在进行的工作，旨在将 OAuth 2.0 及其许多常见扩展整合到一个新名称下。

问题、建议和协议变更应在[邮件列表](https://www.ietf.org/mailman/listinfo/oauth)上讨论。

### OAuth 2.0

- [OAuth 2.0 Framework](https://tools.ietf.org/html/rfc6749) - RFC 6749
	- [Access Tokens](https://oauth.net/2/access-tokens/)
		- [Refresh Tokens](https://oauth.net/2/refresh-tokens/)
		- [OAuth Scope](https://oauth.net/2/scope/)
- [OAuth Grant Types](https://oauth.net/2/grant-types/)
	- [Authorization Code](https://oauth.net/2/grant-types/authorization-code/)
		- [PKCE](https://oauth.net/2/pkce/)
		- [Client Credentials](https://oauth.net/2/grant-types/client-credentials/)
		- [Device Code](https://oauth.net/2/grant-types/device-code/)
		- [Refresh Token](https://oauth.net/2/grant-types/refresh-token/)
		- 已废弃：[Implicit Flow](https://oauth.net/2/grant-types/implicit/)
		- 已废弃：[Password Grant](https://oauth.net/2/grant-types/password/)
- [客户端类型 - Confidential 和 Public 应用](https://oauth.net/2/client-types/)
- [Client Authentication](https://oauth.net/2/client-authentication/)
- [Bearer Tokens](https://oauth.net/2/bearer-tokens/) - RFC 6750
- [威胁模型和安全考量](https://oauth.net/2/security-considerations/) - RFC 6819
- [OAuth 安全最佳实践](https://oauth.net/2/oauth-best-practice/) - RFC 9700
- [ID Token 与 Access Token 对比](https://oauth.net/id-tokens-vs-access-tokens/)

#### 移动设备和其他设备

- [Native Apps](https://oauth.net/2/native-apps/) - RFC 8252，在原生应用中使用 OAuth 的建议
- [Browser-Based Apps](https://oauth.net/2/browser-based-apps/) - 在基于浏览器的应用（如 SPA）中使用 OAuth 的建议
- [Device Authorization Grant](https://oauth.net/2/device-flow/) - RFC 8628，适用于没有浏览器或没有键盘的设备

#### Token 和 Token 管理

- [JWT Profile for Access Tokens](https://oauth.net/2/jwt-access-tokens/) - RFC 9068，结构化 Access Token 的标准
- [Token Introspection](https://oauth.net/2/token-introspection/) - RFC 7662，用于确定 Token 的活跃状态和元信息
- [Token Revocation](https://oauth.net/2/token-revocation/) - RFC 7009，用于通知授权服务器之前获取的 Token 不再需要
- [JSON Web Token](https://oauth.net/2/jwt/) - RFC 7519
- [Token Exchange](https://oauth.net/2/token-exchange/) - RFC 8693

#### 发现和注册

- [Authorization Server Metadata](https://oauth.net/2/authorization-server-metadata/) - RFC 8414，用于客户端发现 OAuth 端点和授权服务器能力
- [Dynamic Client Registration](https://oauth.net/2/dynamic-client-registration/) - RFC 7591，用于以编程方式注册 OAuth 客户端
- [Dynamic Client Registration Management](https://oauth.net/2/dynamic-client-management/) - 实验性 RFC 7592，用于更新和管理动态注册的 OAuth 客户端

#### 高安全级别 OAuth

以下规范用于在 OAuth 2.0 之上增加额外的安全属性。

- [Pushed Authorization Requests (PAR)](https://oauth.net/2/pushed-authorization-requests/) - RFC 9126
- [Demonstration of Proof of Possession (DPoP)](https://oauth.net/2/dpop/) - RFC 9449
- [Mutual TLS](https://oauth.net/2/mtls/) - RFC 8705
- [Private Key JWT](https://oauth.net/private-key-jwt/) - (RFC 7521, OpenID)
- [FAPI](https://oauth.net/fapi/)
